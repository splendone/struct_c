# 单链表结构可视化

## 传统表示方式
```
[数据|指针] -> [数据|指针] -> [数据|指针] -> NULL
```

## 更形象的可视化表示

### 1. 盒子模型表示法
```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│ 数据域: A   │    │ 数据域: B   │    │ 数据域: C   │
│ 指针域: ○───┼───→│ 指针域: ○───┼───→│ 指针域: ○───┼───→ NULL
│ (Data)      │    │ (Data)      │    │ (Data)      │
└─────────────┘    └─────────────┘    └─────────────┘
```

### 2. 分离式表示法
```
数据部分:    [A]     [B]     [C]
            ↓       ↓       ↓
指针部分:   ──→     ──→     ──→ NULL
```

### 3. 节点详细分解
```
节点1: ┌─────┬─────┐
      │  A  │  ●──┼───→ 指向下一个节点
      └─────┴─────┘

节点2: ┌─────┬─────┐
      │  B  │  ●──┼───→ 指向下一个节点  
      └─────┴─────┘

节点3: ┌─────┬─────┐
      │  C  │  ●──┼───→ NULL (结束)
      └─────┴─────┘
```

### 4. 链条比喻表示
```
头节点
  │
  ▼
[数据A|→] ── [数据B|→] ── [数据C|→] ── [NULL]
  数据区      数据区      数据区      结束标志
  指针区      指针区      指针区
```

### 5. 垂直分解图
```
┌─────────────────┐
│    节点 #1      │
├─────────────────┤
│  ┌─────────────┐│
│  │    数据A    ││ ←─ 数据存储区域
│  └─────────────┘│
│  ┌─────────────┐│
│  │    →        ││ ←─ 指向下一个节点的指针
│  └─────────────┘│
└─────────────────┘
         │
         ▼
┌─────────────────┐
│    节点 #2      │
├─────────────────┤
│  ┌─────────────┐│
│  │    数据B    ││ ←─ 数据存储区域
│  └─────────────┘│
│  ┌─────────────┐│
│  │    →        ││ ←─ 指向下一个节点的指针
│  └─────────────┘│
└─────────────────┘
         │
         ▼
┌─────────────────┐
│    节点 #3      │
├─────────────────┤
│  ┌─────────────┐│
│  │    数据C    ││ ←─ 数据存储区域
│  └─────────────┘│
│  ┌─────────────┐│
│  │   NULL      ││ ←─ 终结符，表示链表结束
│  └─────────────┘│
└─────────────────┘
```

### 6. 实际代码结构对应
```c
typedef struct Node {
    int data;           // ┌─────────────┐ ← 数据域
                        // │    数据     │
    struct Node* next;  // ├─────────────┤ ← 指针域
} ListNode;            // │  指向下一节点 │
                        // └─────────────┘
```

这种可视化表示方法更清楚地展示了单链表中每个节点包含两个部分：
- **数据域**：存储实际的数据
- **指针域**：存储指向下一个节点的地址（链接）

通过这种图形化表示，可以更容易理解单链表的结构特点：数据在逻辑上连续，但在物理内存中可能分散存储，通过指针连接形成链式结构。
